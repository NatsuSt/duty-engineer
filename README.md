# Застосунок чергового оператора

Призначення цього застосунку полягає мати постійне нагадування операторам, до кого потрібно звертатися під час виникнення не стандартних ситуацій.

Застосунок завжди знаходиться поверх інших вікон, що дає не загубитися вікну для операторів, при зміні робочого оточення. Застосунок запускається в треї та має можливість змінити оператора на наступного в списку налаштувань та можливість закінчити своє виконання.

Додана можливість виконувати виклик оператора за використанням АТС Asterisk. Для цього в налаштуваннях потрібно додати налаштування під'єднання до AMI-інтерфейсу за використанням правильних налаштувань. 

## Налаштування 

Усі налаштування параметрів програми на даному етапі проєкта полягають у зміні файлу конфігурації, що 
знаходиться за шляхом `%AppData%\Roaming\Serhii\DutyEngineers\config` для систем під керівництвом Windows.
Файл зберігається у форматі `toml`. За замовченням, якщо не існує у місці призначення необхідного файлу,
створюється шаблонний варіант, з одним користувачем. За аналогією якого необхідно додавати інших чергових.

Для налаштування інтерфейсу Asterisk Machine Interface (AMI), потрібно додати об'єкти `ami_manager` та `ami_operator`.

Налаштовуючи менеджер, необхідно вказати порт та хост серверу з Asterisk. На самому сервері
потрібно заздалегідь підготувати користувача, через який буде виконуватися з'єднання.

```
[ami_manager]
port = 5038
host = "asterisk_server_ip_or_name"
username = "usernme"
password = "paassword"
events = true
```

Та для того, щоб до оператора надходив дзвінок, потрібно вказати номер оператора та його контекст.

```
[ami_operator]
operator_number = 111
context = "context"
```
